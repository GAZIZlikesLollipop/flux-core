## 📖 О проекте

**Flux Core** — серверная часть мессенджера Flux. Это намеренно простой и лёгкий WebSocket-сервер написанный на Go, который реализует единственную задачу: **доставить данные от одного клиента к другому**.

Сервер не хранит сообщения, не занимается авторизацией и не содержит бизнес-логики. Вся ответственность за хранение и обработку данных лежит на клиентском приложении [Flux App](../flux-app).

---

## ✨ Принцип работы

```
Клиент A  ──────►  Flux Core  ──────►  Клиент B
           (WebSocket)        (WebSocket)
```

1. Клиент подключается к серверу по WebSocket, передавая свой `User-Id` в заголовке
2. Сервер регистрирует соединение в in-memory кэше
3. При получении сообщения от клиента A, сервер находит соединение клиента B и пересылает данные
4. Если клиент B оффлайн — сервер ожидает его подключения через канал `newCnn`

### Основные компоненты

| Компонент | Описание |
|---|---|
| `Cache` | In-memory хранилище активных WebSocket-соединений (`map[userId]*websocket.Conn`) |
| `Cache.ServeHTTP` | HTTP-хендлер, апгрейдит соединение до WebSocket и обрабатывает сообщения |
| `Request` | Структура входящего сообщения: `receiver_id` + `data` (байты) |
| `Response` | Структура исходящего сообщения: `sender_id` + `data` (байты) |
| `newCnn chan` | Канал для уведомления о новых подключениях (ожидание оффлайн-клиента) |

## 🔌 Протокол

### Подключение

Клиент подключается к `ws://<host>:8080`, передавая свой идентификатор в заголовке:

```
GET ws://localhost:8080
User-Id: <уникальный-id-пользователя>
```

### Формат сообщений

**Запрос (клиент → сервер):**
```json
{
  "receiver_id": "id-получателя",
  "data": [/* байты полезной нагрузки */]
}
```

**Ответ (сервер → клиент-получатель):**
```json
{
  "sender_id": "id-отправителя",
  "data": [/* байты полезной нагрузки */]
}
```

> Поле `data` — произвольный набор байт. Сервер не интерпретирует содержимое, только пересылает его.

### Обработка ошибок

| Ситуация | Ответ сервера |
|---|---|
| Ошибка парсинга JSON | Текстовое сообщение об ошибке, соединение закрывается |
| Попытка написать самому себе | Текстовое сообщение, соединение продолжается |
| Получатель не найден в кэше | Текстовое сообщение, соединение продолжается |
| Ошибка отправки сообщения | Текстовое сообщение об ошибке |

## 🚀 Запуск

### Требования

- Go `>=1.21`

### Установка зависимостей

```bash
go mod download
```

### Запуск сервера

```bash
go run main.go
```

Сервер запустится на `0.0.0.0:8080`.

### Сборка бинарника

```bash
go build -o flux-core main.go
./flux-core
```

### Docker (опционально)

```dockerfile
FROM golang:1.26-alpine AS builder
WORKDIR /app
COPY . .
RUN go build -o flux-core main.go

FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/flux-core .
EXPOSE 8080
CMD ["./flux-core"]
```

```bash
docker build -t flux-core .
docker run -p 8080:8080 flux-core
```
